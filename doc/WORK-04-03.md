# Hackathon Work Summary

## 1. 작업 개요

이번 작업에서는 `Backend`의 커뮤니티 기능용 DB 마이그레이션과 mock 데이터를 정리하고, `Frontend`에 커뮤니티 진입 및 조회 화면을 추가했다.

핵심 목표는 다음 3가지였다.

1. `community_mock.sql` 내용을 현재 백엔드 엔티티 구조와 맞는지 검토
2. Flyway 실행 시 커뮤니티 관련 테이블과 mock 데이터가 자동으로 들어가도록 구성
3. 프론트 홈에서 커뮤니티로 진입할 수 있게 하고, 커뮤니티 목록/상세 조회 UI를 연결

---

## 2. Backend 변경 사항

### 2-1. Flyway 마이그레이션 추가

다음 파일을 추가/수정했다.

- `Backend/src/main/resources/db/migration/V6__create_community_tables.sql`
- `Backend/src/main/resources/db/migration/V7__insert_community_mock_data.sql`

적용 내용:

- `community_post` 테이블 생성
- `comment` 테이블 생성
- 현재 엔티티 기준으로 `comment.language` 컬럼 포함
- `community_post.tag`는 문자열 컬럼으로 반영
- 게시글 20개, 댓글 40개의 커뮤니티 mock 데이터 삽입

### 2-2. 엔티티 기준 불일치 정리

초기 `community_mock.sql`에서 확인된 문제:

- `post_id` 사용
- `translated_title` 사용
- enum에 없는 카테고리 값 사용
- 기존 user seed와 충돌 가능성 존재

이를 기준으로 실제 실행용 데이터는 `V7__insert_community_mock_data.sql`에 맞춰 재정리했다.

### 2-3. RestClient 설정 추가

다음 파일을 추가했다.

- `Backend/src/main/java/com/carelink/global/config/RestClientConfig.java`

추가 이유:

- `GeminiTranslationClient`가 `RestClient.Builder`를 주입받고 있었는데 빈이 없어서 앱 기동이 실패했음
- `RestClient.Builder`를 명시적으로 빈 등록해서 해당 오류를 해결함

### 2-4. 확인 결과

확인 완료:

- Flyway `V1 ~ V7` 정상 적용
- 커뮤니티 테이블 생성 및 데이터 적재 성공
- `./gradlew compileJava --no-daemon` 성공

현재 남아 있는 런타임 이슈:

- `GEMINI_API_KEY`가 없으면 앱 기동 실패
- 즉, DB/Flyway 문제는 해결되었고 현재 남은 문제는 환경변수 설정 문제임

관련 에러:

- `Could not resolve placeholder 'GEMINI_API_KEY' in value "${GEMINI_API_KEY}"`

필요 환경변수 예시:

```env
GEMINI_API_KEY=your_key
GEMINI_MODEL=gemini-2.5-flash
```

## 3. Frontend 변경 사항

### 3-1. 커뮤니티 API/타입 추가

추가 파일:

- `Frontend/src/types/community.ts`
- `Frontend/src/api/communityApi.ts`

구현 내용:

- 커뮤니티 게시글 목록 조회
- 커뮤니티 게시글 단건 조회
- 게시글별 댓글 조회

### 3-2. 커뮤니티 화면 추가

추가 파일:

- `Frontend/src/components/community/CommunityPage.tsx`
- `Frontend/src/components/community/CommunityDetailPage.tsx`

구현 내용:

- 커뮤니티 목록 페이지
- 상단 카테고리 필터
  - `ALL`
  - `NOTICE`
  - `QUESTION`
  - `REVIEW`
  - `FREE`
- 게시글 클릭 시 상세 페이지 이동
- 상세 페이지에서 본문 + 댓글 조회

### 3-3. 라우팅 추가

수정 파일:

- `Frontend/src/App.tsx`

추가 라우트:

- `/community`
- `/community/:postId`

### 3-4. 홈 화면 연동

수정 파일:

- `Frontend/src/components/home/MainHomeScreen.tsx`

반영 내용:

- 홈 디자인은 유지
- 기존 하단 커뮤니티 버튼을 실제 `/community` 링크로 연결
